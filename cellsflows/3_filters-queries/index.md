Filters and Queries modules are the components used to either filter a message content or dynamically load contents from Cells internals data to feed Actions inputs. 

As explained in the [Data Types](./data-types-queries-filters) section, these blocks are "typed" and use combination of criteria to build filtering or loading conditions. Below are describe the low-level blocks used for building conditions. When they are named "Selectors", it means that they can be used either in Filters or Queries components.

- [DataSource Selector](/cells-v4/cellsflows/filters-queries/datasource-selector/)
- [Files/Folders Selector](/cells-v4/cellsflows/filters-queries/files-folders-selector/)
- [Users Selector](/cells-v4/cellsflows/filters-queries/users-selector/)
- [Roles Selector](/cells-v4/cellsflows/filters-queries/roles-selector/)
- [Workspaces Selector](/cells-v4/cellsflows/filters-queries/workspaces-selector/)
- [ACL Selector](/cells-v4/cellsflows/filters-queries/acl-selector/)
- [Task Message Filter](/cells-v4/cellsflows/filters-queries/task-message-filter/)
- [Request Metadata Filter](/cells-v4/cellsflows/filters-queries/request-metadata-filter/)
- [Arbitrary Data Selector](/cells-v4/cellsflows/filters-queries/arbitrary-data-selector/)
- [Job Trigger Filter](/cells-v4/cellsflows/filters-queries/job-trigger-filter/)
